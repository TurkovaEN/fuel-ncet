# Fuel NCET Legacy (Обоснование НМЦК по топливу)

Десктопное Windows‑приложение с графическим интерфейсом для расчёта **обоснования начальной цены единицы товара** (бензин/дизель) по данным Росстата и формирования итогового документа **.docx** по шаблону Word.

Версия проекта адаптирована под запуск на старых системах (в т.ч. Windows Server 2008 R2), поэтому используется Python 3.8 и сборка через PyInstaller.

## Возможности
- Графический интерфейс (без консоли).
- Автозагрузка цен с сайта **22.rosstat.gov.ru**:
  - поиск публикации *«Потребительские цены на …»*;
  - выбор последней подходящей публикации **не позже указанной даты формирования**;
  - поддержка перехода по страницам списка новостей (пагинация «Далее/Назад»), а не только первой страницы;
  - скачивание PDF по региону **Алтайский край**;
  - извлечение значений по колонке **Барнаул** для:
    - Дизельное топливо, л
    - Бензин АИ‑92, л
    - Бензин АИ‑95, л
- Поле **«Дата формирования»** (по умолчанию текущая) — можно менять; по этой дате выбираются последние доступные данные Росстата.
- Расчёт:
  - ежемесячного индекса `m` из годового индекса прогнозной инфляции (ввод вручную, напр. `104,0%`);
  - дефляторов на начало/конец поставки по принятой схеме степеней (есть ручная корректировка степеней `n_start/n_end`);
  - ИПЦ на период поставки (среднее значение);
  - итоговых сумм по таблице.
- Предпросмотр таблицы в окне программы.
- Экспорт итогового документа **DOCX** по шаблону `assets/template.docx`.
- Числа в строках “прописью” (рубли/копейки) с правильными склонениями.
- Кэширование скачанного PDF Росстата в `cache/` (опционально очистка кэша при выходе).
- Кнопка **«Сброс»** — очищает все поля до начального состояния.
- При проблемах SSL на старых системах загрузчик может повторить запрос без проверки сертификата; в интерфейсе это отображается как “SSL без проверки”.

## Требования

### Для разработки / сборки
- Windows
- Python 3.8.x
- Git (опционально)

### Для запуска готового `.exe`
- Если сборка сделана как `--onefile` — можно запускать один `.exe`.
- Если сборка сделана как `--onedir` — нужно переносить и запускать **всю папку** из `dist/` (запускать основной `.exe` внутри неё).

## Сборка в `.exe` (PyInstaller)

Пример сборки (legacy, `--onedir`):

```powershell
.\.venv38\Scripts\python.exe -m PyInstaller `
  --noconfirm --clean `
  --onedir --windowed `
  --name "FuelNCET_Legacy" `
  --paths "src" `
  --icon "assets\app.ico" `
  --add-data "assets\template.docx;assets" `
  --add-data "assets\app.ico;assets" `
  main.py
